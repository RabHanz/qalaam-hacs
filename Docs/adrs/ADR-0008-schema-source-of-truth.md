# ADR-0008: JSON Schema as single source of truth → codegen TS + Python

- **Status:** Accepted
- **Date:** 2026-05-02
- **Deciders:** Founder
- **Outcome served:** All (type safety reduces all bug classes; foundational)
- **Consulted:** N/A
- **Informed:** Future contributors

## Context

Qalaam ships TypeScript packages (frontend, backend, web adapters, HA-as-adapter shim) and Python packages (HA integration, ASR worker, TTS worker, device-bridge sidecars). Both languages must agree on the shape of every shared entity: `Verse`, `Reciter`, `AudioSegment`, `Speaker`, `Adapter`, `Plan`, `Portion`, `ReviewState`, `RatingEvent`, `MistakeEvent`, `MutashabihatCluster`, etc.

Without a single source of truth, drift is inevitable: a TS type adds a field, the Python Pydantic model doesn't, the integration silently breaks at the boundary. CLAUDE.md §11 Code Quality non-negotiables ("TypeScript strict mode or equivalent type safety in other languages") demands this be solved architecturally.

## Decision

We will define every cross-language entity in **JSON Schema (Draft 2020-12)** in `packages/schema/schemas/`. A codegen pipeline (`pnpm codegen` + `uv run codegen`) produces:
- `packages/types-ts/dist/` — TypeScript types via `json-schema-to-typescript`
- `packages/types-py/qalaam_types/` — Pydantic v2 models via `datamodel-code-generator`

Generated files are committed to source control. CI gate: `pnpm codegen && git diff --exit-code` — fails if generated artifacts are out of sync with schemas.

OpenAPI specs for HTTP endpoints are also generated from the same schemas (via TypeBox or equivalent in the Fastify backend), so `api-client-ts` and `api-client-py` stay in lockstep.

**Privacy-architectural enforcement:** the cloud-sync transport schemas (`schemas/api/cloud-sync-*.schema.json`) explicitly forbid audio-bearing fields at the schema level (per ADR-0005 privacy posture). Audio types only appear in `schemas/asr/local-only-*.schema.json` which are never used by cloud transport.

## Alternatives considered

1. **Hand-written types in each language.** **Rejected because** drift is inevitable; CLAUDE.md non-negotiable for type safety.

2. **TypeScript as source of truth, Python types derived.** Considered. **Rejected because** TS types alone cannot express runtime validation rules (regex patterns, ranges) that JSON Schema can.

3. **Protocol Buffers / Cap'n Proto.** **Rejected because** the wire-format benefits don't justify the tooling overhead for our HTTP-first surface; JSON over the wire is the default.

4. **OpenAPI as source of truth.** Considered. **Rejected because** OpenAPI is HTTP-shaped, but we need types that travel over WebSocket, gRPC, and in-memory too. JSON Schema is the broader primitive; OpenAPI references it.

5. **Status quo (no codegen).** **Rejected** as above.

## Consequences

### Positive

- Adding a field to `MistakeEvent` requires editing one file; downstream PRs in TS and Python are auto-generated.
- Privacy guarantee (no audio in cloud sync) is enforced at the type level — not even possible to write the wrong code.
- API specs (OpenAPI) auto-derive from the same source — single source of truth across schema + types + API contract.
- Onboarding clarity: "all data shapes live in `packages/schema`."

### Negative

- Codegen step in the dev loop. Mitigation: `pnpm dev` watches and re-runs.
- Two codegen toolchains (`json-schema-to-typescript` for TS, `datamodel-code-generator` for Python) — small surface but two upgrade paths.
- Generated Python Pydantic v2 models occasionally need post-processing for advanced JSON Schema features.

### Neutral

- Generated files in source control means PR diffs include `dist/`. We accept this trade-off for explicit visibility into what changed.

## Risks & monitoring

- **Risk:** Codegen drifts (someone edits a generated file by hand). **Leading indicator:** CI gate fails. **Mitigation:** CI gate is non-bypassable; generated dirs include `// THIS FILE IS AUTOGENERATED. DO NOT EDIT.` headers.
- **Risk:** Codegen tools (`datamodel-code-generator`, `json-schema-to-typescript`) have a regression. **Leading indicator:** snapshot test failures in `packages/schema/tests/`. **Mitigation:** version-pin both tools.
- **Risk:** A schema change that should be a backward-incompatible major bump is shipped as a minor. **Leading indicator:** downstream package CI breaks unexpectedly. **Mitigation:** Changesets-driven semver review per package.
